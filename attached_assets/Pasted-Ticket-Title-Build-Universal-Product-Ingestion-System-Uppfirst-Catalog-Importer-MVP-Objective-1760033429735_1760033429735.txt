Ticket Title:
Build Universal Product Ingestion System (Uppfirst Catalog Importer MVP)

⸻

Objective:
Enable mass import and sync of seller catalogs from Shopify, WooCommerce, BigCommerce, and Etsy into Uppfirst without manual mapping.

⸻

Scope & Deliverables
	1.	Core Infrastructure
	•	Backend service (Node/TS + Postgres + Redis)
	•	Canonical schema for Product, Variant, Price, Inventory, Media, Collection
	•	Bulk import pipeline (chunked jobs, async queue)
	•	Idempotent upsert logic (source + source_id key)
	•	Validation (required fields, price > 0, safe HTML)
	•	Cursor-based pagination & checkpointing
	•	Image handling (store URLs, no hosting initially)
	•	Basic admin panel (jobs view, replay failed, progress %)
	•	Webhooks + delta sync for real-time updates
	2.	Adapters (Phase 1)
	•	Shopify: Bulk GraphQL + webhooks
	•	WooCommerce: REST v3 + product webhooks
	•	BigCommerce: Catalog v3 + webhook events
	•	Etsy: REST listings API + modified_since fallback
	3.	Data Flow
	•	OAuth2 (or API keys) for auth per platform
	•	Background importer → normalize → store in DB → expose via Uppfirst API
	•	Queue-based async job processing
	•	Webhook ingestion for deltas; nightly full resync
	4.	Testing & QA
	•	Mock datasets from each source
	•	Load tests for 100k SKUs
	•	Delta test: modify one product → update reflected
	•	Failover test: partial import retry
	5.	Observability
	•	Structured logs (per job, per source)
	•	Metrics (import time, failures, rate-limit hits)
	•	Dead-letter queue + replay button
	6.	Security
	•	Encrypted API credentials (Vault or env store)
	•	OAuth token refresh management
	•	Signed webhook verification

⸻

Timeline:
~105 dev-days total
	•	Core system: 40
	•	Shopify: 13
	•	WooCommerce: 10
	•	BigCommerce: 10
	•	Etsy: 15
	•	QA + buffer: 17

⸻

Dependencies:
	•	Existing Uppfirst Auth + User DB
	•	API gateway or internal route for /catalog/import
	•	Redis / Queue infra

⸻

Acceptance Criteria
	•	Seller connects store → system imports full catalog within 1 h (≤50k SKUs).
	•	Incremental sync latency < 5 min (Shopify/Woo).
	•	No duplicate or missing SKUs after two full runs.
	•	Failed jobs retry automatically.
